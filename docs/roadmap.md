# Roadmap

The roadmap communicates priorities, not promises. New work should preserve local-first operation, one source of truth, and honest failure states.

## Near term

- Unified task timelines, failure causes, retries, and diagnostics.
- Stronger cost, health, expiry-risk, and grouping views.
- Safer import previews and clearer batch outcomes.
- Automated desktop builds, checksums, and release notes.
- More operating examples and troubleshooting guides.

## Extension points

- **Data-source adapters** with explicit provenance and snapshot semantics.
- **Notification adapters** for common collaboration tools without exposing keys to the browser.
- **Readable rules** for price, expiry, and quota conditions.
- **Encrypted backup and restore** with clear separation between public settings and secrets.

## Longer-term research

Multi-device read-only views, optional remote control, granular access, plugin runtimes, and cross-platform auto-update all change the current trust boundary. Each requires a threat model, migration plan, and architecture decision record before implementation.
