# 后期扩展

路线图用于公开优先级，不是功能承诺。新能力应继续遵守本地优先、事实源唯一和失败状态诚实三个原则。

## 近期

- 统一任务时间线、失败原因、重试和诊断。
- 完善成本、健康度、到期风险和账户分组。
- 增加安全导入预检和批量结果汇总。
- 自动化桌面构建、校验和版本说明。
- 补齐操作示例与故障排查。

## 可扩展方向

- 带来源标识和快照语义的数据源适配器。
- 不向浏览器暴露密钥的通知适配器。
- 面向价格、到期与额度条件的可读规则。
- 明确区分公开设置与敏感材料的加密备份。

## 长期研究

多设备只读视图、远端控制、细粒度权限、插件运行时和跨平台自动更新都会改变当前信任边界。实现前需要威胁模型、迁移方案和架构决策记录。
