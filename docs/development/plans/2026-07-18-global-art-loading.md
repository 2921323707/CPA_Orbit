# 全局艺术加载动画设计

## 目标

让 CPA Monitor 在首屏和路由切换时先稳定呈现导航，再逐步呈现数据内容；动效需要克制、低对比、可识别，并且不影响操作效率。

## 方案

- `AppShell` 始终先渲染侧栏与顶部栏。
- 路由开始切换时显示主内容级加载层，使用同心轨道、十字刻度、扫描线和信号点组成的观测仪表，不使用通用转圈。
- 路由完成后保留短暂缓冲，再以 `page-reveal` 和子元素错峰动画显示页面。
- 各页面已有的数据加载状态复用同一套艺术加载组件的紧凑版本。
- `prefers-reduced-motion` 时由全局 CSS 将动画和过渡降至近乎静止。

## 运行边界

加载层只遮挡主内容，不遮挡导航、服务状态或主题切换；接口失败仍由页面原有错误状态接管，不会无限等待。
