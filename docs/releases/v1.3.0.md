# CPA Orbit v1.3.0

**Released:** July 22, 2026

**Release title:** Explicit Gateway Assignment Control Plane

CPA Orbit v1.3.0 develops a durable local control plane for independently configured CPA and Sub2API targets. Each new Auth JSON import explicitly selects exactly one compatible target. CPA may be selected as an alternate, migration, or recovery destination, but a failed or uncertain Sub2API import is never automatically switched to CPA.

## Highlights

- Adds a two-stage flow: bounded local Auth JSON preflight followed by an explicit commit to one target.
- Preserves provider/date subscription archives independently from remote deployment success.
- Records gateway targets, managed/adopted ownership, desired/observed bindings, import outcomes, operation history, and aggregate usage in local SQLite.
- Distinguishes definite deployment failures from uncertain transport outcomes and exposes sanitized error code, target, HTTP status, and retryability.
- Resumes a retryable import through the same durable operation, archive, and target instead of creating a duplicate or switching gateways.
- Enforces one active pool assignment for refreshable credentials and supports explicit detach/deploy migration.
- Safely removes only remote accounts explicitly created by Orbit; adopted accounts are unbound without remote deletion.
- Moves gateway health and configuration into **Settings → Gateways**; the superseded dedicated operations route is not part of the final navigation.
- Stores normalized 15-minute request, Token, latency, and cost aggregates for up to 90 days while leaving raw logs in Sub2API.
- Polls account status and quota independently from offer monitoring, every five minutes by default; interval `0` disables scheduled polling.
- Supports the current Sub2API SSE account-test contract, preserves the distinction between account health and transient inspection availability, and displays measured latency and quota without false abnormalities.
- Safely reconciles a stale Sub2API remote account ID only through a unique Orbit-provenance and strong-identity match; ambiguity never triggers a rebind, redeploy, or delete.
- Streamlines the subscription table by removing its crowded action column; the subscription filename opens the detail drawer where account and binding actions remain available.

## External Sub2API setup

CPA Orbit does not include, install, or download Sub2API. Run it separately—for example with Docker—and keep the management endpoint on a trusted interface. For a local deployment exposed at `http://127.0.0.1:8080`, create an administrator key under **Sub2API System Settings → Admin API Key**, then add that endpoint and key under **CPA Orbit Settings → Gateways** and run **Check connection**.

Official CPA Orbit EXE, DMG, and ZIP artifacts embed neither Sub2API nor CLIProxyAPI.

## Security boundaries

Sub2API administrator keys are write-only. Remote management endpoints require explicit enablement and HTTPS; redirects and oversized responses are rejected. Errors and public models exclude credential content. A primary marker does not enable automatic cross-target failover.

Subscription-to-gateway conversion can conflict with upstream provider terms and may lead to account suspension or interruption. Use only where authorized and review the upstream agreements before deployment.
