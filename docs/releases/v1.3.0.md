# CPA Orbit v1.3.0

**Release title:** Sub2API Subscription Pool Control Plane

CPA Orbit v1.3.0 adds a durable local control plane for Sub2API and CPA. Sub2API becomes the preferred primary subscription pool, while CPA remains a lightweight deployment fallback and recovery target.

## Highlights

- Imports GPT Plus/Codex session JSON through the current Sub2API administrator API.
- Preserves `k12/` archives independently from runtime deployment success.
- Records gateway targets, managed/adopted ownership, desired/observed bindings, and operation history in local SQLite.
- Enforces one active primary binding for refreshable credentials and supports detach/deploy migration with rollback.
- Safely removes only remote accounts explicitly created by Orbit.
- Adds a pool operations console with health, bindings, operations, live Sub2API totals, manual refresh, and seven-day Token charts.
- Stores normalized 15-minute request, Token, latency, and cost aggregates for up to 90 days while leaving raw logs in Sub2API.

## Security boundaries

Sub2API administrator keys are write-only. Remote management endpoints require explicit enablement and HTTPS; redirects and oversized responses are rejected. Errors and public models exclude credential content.

Subscription-to-gateway conversion can conflict with upstream provider terms and may lead to account suspension or interruption. Use only where authorized and review the upstream agreements before deployment.

## Windows artifact

The portable executable is built as `CPAOrbit.exe` under `app/build/bin/`. Verify the SHA-256 value generated during release preparation before redistribution.
