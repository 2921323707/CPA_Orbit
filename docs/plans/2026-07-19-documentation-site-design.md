# CPA Orbit 文档站设计

## 目标

为 CPA Orbit 建立一个可独立部署的中文静态文档站，首版覆盖简介、快速开始、模块介绍、后期扩展、参与贡献与赞助。站点应从现有仓库文档取材，避免形成第二套互相漂移的内容系统。

## 技术方案

采用 VitePress，并直接以 `docs/` 作为内容根目录。Markdown 是内容源，VitePress 提供静态构建、导航、本地搜索、深浅主题与可访问性基础。构建产物为纯 HTML/CSS/JavaScript，可由 Nginx、Caddy、对象存储或静态托管平台直接提供。

相比单独手写 Vue 单页，这一方案更适合文档的长期分层与检索；相比引入数据库和 CMS，它保留了 Git 审查、可移植性和零服务端依赖。

## 信息架构

- 首页：项目定位、核心价值、快速启动和行动入口。
- 快速开始：环境、启动、状态确认、CPA 配置和首份订阅导入。
- 模块介绍：价格、订阅、CPA 运行、提醒、短信和桌面宿主。
- 后期扩展：近期、中期、长期方向及进入实现前的约束。
- 贡献：入口、开发流程、检查清单和工程原则。
- 赞助：资金用途、独立性承诺和待上线渠道。
- 深入参考：复用现有架构、后端、桌面、发布与安全文档。

## 视觉方向

视觉采用“编辑部式技术档案”：暖灰纸张底色、墨绿主色、少量陶土橙作为信号色。标题使用衬线字体栈，正文使用中文无衬线字体栈，代码使用等宽字体。首页保留一个克制的轨道几何符号，不使用大面积渐变、玻璃卡片或装饰性插画。

组件以 1px 细边框、紧凑圆角和清晰编号组织信息。动效仅用于悬停反馈，并遵守 `prefers-reduced-motion`。

## 验证

生产构建必须成功，内部链接由 VitePress 在构建期间解析。首页、内容页、导航、搜索和深浅主题需在移动端与桌面端保持可读；部署说明以静态产物目录为唯一交付边界。
